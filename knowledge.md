# 第二周学习知识记录

## Query Transformation
Query Transform（查询转换） 是 RAG（检索增强生成）系统中的关键技术，用于优化用户原始查询，使其更适合向量检索，从而提高检索质量和最终答案的准确性。

**用户的原始查询往往存在以下问题**：
过于简短或模糊（如"怎么优化？"）
包含错别字或语法错误
使用口语化表达而非专业术语
复杂多跳问题（需要多个步骤才能回答）
指代不清（依赖上下文中的代词）
直接将这些查询向量化，常常导致检索到不相关的文档。

### 主要技术手段
#### MultiQuery Retrieval
原理：LLM 为一个问题生成多个不同表达方式。

例如：
用户问题：
什么是LangChain？

LLM 生成：
LangChain是什么
LangChain框架介绍
LangChain的作用

然后：
三个 query 分别检索
↓
合并结果
优点：
提升 Recall（召回率）
LLM 为一个问题生成多个不同表达方式。

####  Query Rewriting（查询重写）
使用 LLM 重新表述查询，使其更清晰、完整.
将口语化表达转换为专业术语
示例：
原始："RAG 性能问题"
重写："检索增强生成（RAG）系统的延迟优化、向量搜索性能调优方法"

#### HyDE（假设文档嵌入）
让 LLM 先生成一个假设的答案/文档，然后用这个假设文档的嵌入去检索，而不是用原始查询。因为假设文档的嵌入空间更接近真实文档。
文档式语义比问题式语义更接近知识库内容
示例：
查询："如何申请育儿假？"
HyDE 生成："申请育儿假需要提交申请表、提供预产期证明、提前30天通知HR..."
用这个生成内容去检索相关文档

User Query
   │
   ▼
LLM 生成一个“假想答案”(Hypothetical Answer)
   │
   ▼
对这个假想答案做 Embedding
   │
   ▼
向量检索
   │
   ▼
找到真正相关文档

#### Query Expansion（查询扩展）
通过添加相关术语、同义词或概念来扩大搜索范围：
词汇扩展：添加同义词、相关实体
伪答案生成：生成可能的答案片段作为扩展
多查询扩展（Multi-Query）：生成 3-5 个不同角度的查询变体，并行检索后合并结果

#### Query Decomposition（查询分解）
将复杂查询拆分为多个简单子查询：适用于多跳问题（如比较类、因果类问题）
示例：
原始："LangChain 中上下文窗口管理和持久化内存存储有何区别？"
分解为：
"LangChain 有哪些上下文窗口管理策略？"
"LangChain 有哪些持久化内存存储方案？"
"两者的主要区别是什么？"

Query Transform 是低成本高收益的优化手段——不需要更换向量数据库或重训嵌入模型，仅需在检索前增加一个 LLM 调用层，通常可带来 25-40% 的精度提升

**选择MultiQuery Retrieval进行本周实现**
原因：
实现简单
不容易引入噪声
对所有 embedding 模型都有效
提升 recall 最大
工业界常用
最稳定


## 混合检索与重排 (Rerank)
### BM25（Best Match 25） 
关键词检索算法，属于稀疏检索方法。根据词项在文档中的出现频率和逆文档频率计算相关性分数。
一个排序函数

**优点**
精确匹配能力强，对关键词敏感
计算速度快，无需神经网络推理
可解释性强，分数可拆解分析	
对短查询（关键词搜索）效果极佳	
参数少，调参简单
**缺点**
无法理解语义，"电脑"和"计算机"视为不同词
对拼写错误、同义词、多语言无能为力
需要维护倒排索引，占用内存
无法捕捉词序和上下文信息
对新词、OOV（未登录词）处理困难

### Reranker
Reranker（重排序器）是信息检索两阶段架构中的第二阶段精排模型。它的任务是在第一阶段（召回）返回的候选文档集合上，进行更精细、更准确的相关性排序。

**Reranker 的优势**
查询和文档同时输入模型，进行深度交互计算
可以捕捉词级别、短语级别的匹配关系
精度显著高于双塔模型，但计算成本高

#### Reranker架构类型
##### Cross-Encoder（交叉编码器）
[CLS] 查询文本 [SEP] 文档文本 [SEP]
         ↓
    BERT/RoBERTa 等预训练模型
         ↓
    [CLS] 位置输出向量
         ↓
    全连接层 → 相关性分数（0-1）

特点：
查询和文档拼接后一次性编码
自注意力机制让查询和文档的每个词都能互相交互
精度最高，但计算成本大（每次查询-文档对都需要前向传播）

##### 双塔模型（对比）
查询文本 → Encoder_q → 向量 v_q
文档文本 → Encoder_d → 向量 v_d
相似度 = cosine(v_q, v_d)

特点：
查询和文档独立编码
文档向量可离线预计算并索引
在线只需编码查询，计算向量相似度
精度较低，但速度快（毫秒级）

#### Reranker优化策略
##### 截断策略
不是对所有文档 rerank，而是对第一阶段 Top-K（如 100-200）进行
平衡精度和延迟

##### 缓存策略
对高频查询的 rerank 结果缓存
对热门文档的向量表示缓存

##### 级联Rerank
先用轻量级 Reranker 筛选到 Top-20
再用重量级 Reranker 精排到 Top-5
类似推荐系统的多级排序

### BM25 + Embedding 混合检索
优势：
互补性：BM25 保证精确性，Embedding 保证语义性
鲁棒性：一种方法失效时，另一种可以补救
全面性：覆盖不同类型的用户查询（关键词型 vs 语义型）

**融合**使用RRF算法

#### 混合检索的完整架构
用户查询
    │
    ├──→ [BM25 检索] ──→ 候选集 A (Top-50)
    │                      │
    ├──→ [向量检索] ───→ 候选集 B (Top-50)  
    │                      │
    │                   [RRF 融合]
    │                      │
    │                   候选集 C (Top-100)
    │                      │
    └────────────────→ [Reranker 精排]
                           │
                        最终结果 (Top-10)

### 注意
Reranker 不是 BM25 + Embedding，它们是三个不同的组件，各自有独立的职责。
